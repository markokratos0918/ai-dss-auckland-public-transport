# Layer 3: Decision Engine

Architecture position:

```text
AI-DSS Modeling / SHAP -> Decision Engine -> SUMO
```

## Purpose

This layer converts delay-risk categories into operational recommendation summaries for review, reporting, SUMO scenario preparation, and the later Streamlit dashboard.

## Required Local Input

```text
data/processed/decision_engine_output.csv
```

This file is generated by Notebook 09 and must remain local because it is a large row-level artifact.

## Command

```bash
python src/audit_decision_engine_outputs.py
```

## Expected Small Outputs

```text
data/processed/summaries/decision_recommendation_summary.csv
data/processed/summaries/decision_route_recommendation_counts.csv
```

Supporting rule table:

```text
data/processed/intervention_logic.csv
```

## Audit Purpose

The audit checks that delay-risk values map consistently to operational recommendations:

| Delay risk | Recommendation role |
| --- | --- |
| Low | No operational action |
| Medium | Monitor route conditions |
| High | Adjust service headway |
| Severe | Standby bus or supervisor review |

## Scope Limits

- The Decision Engine is a rule-based decision-support layer.
- It does not automate public transport operations.
- It should be interpreted as recommendation evidence for the prototype dashboard and report.
