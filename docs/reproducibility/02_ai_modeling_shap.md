# Layer 2: AI-DSS Modeling / SHAP

Architecture position:

```text
Notebook 09 GTFS + Weather Validation
-> AI-DSS Modeling / SHAP
-> Decision Engine
```

## Purpose

This layer evaluates the AI-DSS modeling checkpoint using the official current model-baseline decision output generated by Notebook 09.

The AI model predicts actionable delay risk and delay minutes. The AI evidence is not only SHAP. It includes model metrics, prediction samples, full AI prediction Parquet outputs, and SHAP or feature-importance evidence.

The checkpoint supports:

- XGBoost classification for actionable delay risk.
- XGBoost regression for delay minutes.
- Baseline comparisons.
- ARIMA hourly delay baseline.
- SHAP explainability or model-native feature importance.

## Plain-Language Code Narrative

The AI layer is split into small files so the project is easier to explain, audit, and maintain.

| File | Simple explanation |
|---|---|
| `src/train_ai_dss_model.py` | The start button for the AI modelling program. |
| `src/ai_dss_modeling/config.py` | The settings file: input paths, output paths, columns, features, sample sizes, and other modelling settings. |
| `src/ai_dss_modeling/data.py` | Loads and prepares the Auckland data, then creates the train/test split. |
| `src/ai_dss_modeling/models.py` | Runs the machine learning models and baselines. |
| `src/ai_dss_modeling/metrics.py` | Scores how well the models performed. |
| `src/ai_dss_modeling/explainability.py` | Explains which features influenced the model. |
| `src/ai_dss_modeling/reporting.py` | Saves the final outputs for documentation and dashboard use. |

Simple flow:

```text
train_ai_dss_model.py starts everything
config.py tells it what settings to use
data.py loads and prepares the Auckland data
models.py trains the baseline, XGBoost, regression, and ARIMA models
metrics.py scores the results
explainability.py explains the important features
reporting.py saves the outputs for documentation and dashboard use
```

Notebook 04 was a prototype demonstration using a small Kaggle dataset. The current AI-DSS module is a repeatable project layer that can be rerun with one command and can produce consistent outputs for the Decision Engine, documentation, and final Streamlit dashboard.

## Commands

Run the full modeling checkpoint:

```bash
python src/train_ai_dss_model.py
```

Run a no-write sanity check:

```bash
python src/train_ai_dss_model.py --dry-run
```

## Assessment 1 Alignment

This layer supports:

## Scope Limits

- The current AI model is a prototype checkpoint, not a deployed live predictor.
- The model is evaluated on the official current model-baseline folder.
- High and Severe classes are rare, so performance should not be described as highly accurate overall.
- Weather is included, but route, time, and service features appear stronger in the current dataset.
- Older duplicate root-level decision outputs should not be used.
- Recommendations remain decision-support evidence, not automated operational control.
